## HTML 빌더 4종 모드 — 단일 진실

`output_format: html` 케이스(또는 generic_case에서 사용자가 HTML 요청)는 빌더 모드가 4종이다. 모드는 케이스 프론트매터의 `html_mode` 필드가 단일 진실 — `slides | landing | direct | assisted`. SKILL.md Phase 2는 이 필드를 그대로 참조한다.

| 모드 | 적용 | 슬라이드 템플릿 | BLACK 자체 디자인 |
|---|---|---|---|
| `slides` | `html_mode: slides`인 케이스 (현행 p41·p42·p43·p45) | 35종 중 1개 자동 선택 (formality 高) | 금지 (템플릿 그대로) |
| `landing` | `html_mode: landing`인 케이스 (현행 p70) | 35종 중 1개 자동 선택 (마케팅 색감) | 금지 (템플릿 그대로) |
| `direct` | `html_mode: direct`인 케이스 (현행 p73) | 미사용 | 필수 (자체 디자인 시스템) |
| `assisted` | generic_case + HTML 요청 / `html_mode: assisted`인 케이스 | 35종 중 1개 자동 선택 | **베이스 위에 콘텐츠·인터랙션 보강 허용** |

### `slides` 모드 — PPT 4종 (p41/42/43/45)

- 외부: slide_library의 선택된 템플릿 HTML 그대로
- 콘텐츠 주입: 템플릿의 슬라이드 영역에 BLACK 산출물 삽입 (H1 단위, fallback H2)
- 출력: 단일 `.html` 파일, 브라우저로 발표 가능, PDF 익스포트 가능
- BLACK 자체 디자인 시스템 생성 금지

### `landing` 모드 — p70 웹사이트·랜딩페이지

- 외부: slide_library 35개 템플릿 중 **톤·formality·color로 가장 적정한 1개를 그대로 사용** (PPT와 동일 선택 흐름). 자체 디자인 생성 금지.
- BLACK 산출물의 섹션(히어로·소셜프루프·기능·증거·CTA·FAQ·푸터)을 템플릿의 슬라이드 컨테이너에 H2 단위로 순서대로 주입.
- 출력: 단일 `.html` 파일, 모바일 반응형, CTA 동일 라벨·색.

### `direct` 모드 — p73 웹사이트 제작

- 외부 템플릿 사용 안 함. BLACK이 단일 HTML 파일을 처음부터 직접 작성.
- 인라인 CSS, 시맨틱 마크업(`h1` 1개·섹션별 `h2`·`alt` 필수), 반응형, WCAG 2.1 AA 대비 4.5:1.
- 섹션 4~6개. 히어로 헤드라인 15자 이내. fold 안에 CTA.
- 출력: BLACK 작성 HTML 텍스트를 그대로 `output.html`로 저장. 빌더는 텍스트 통과 외에 아무 작업도 하지 않는다.

### `assisted` 모드 — generic_case의 HTML 빌더 (신설 v0.4)

**적용 조건:** 다음 중 하나
1. Phase 1에서 generic_case로 분류되었고 사용자 입력이 HTML/페이지/대시보드/일정표/카드 등 "디자인이 있는 결과물"을 요구
2. 향후 신규 케이스가 프론트매터에 `html_mode: assisted`를 선언

**핵심 원칙:** BLACK은 처음부터 디자인을 만들지 않는다 (그러면 `direct`와 동일). 슬라이드 라이브러리 템플릿 1개를 베이스로 잡고, 그 위에 **콘텐츠 주입 + 필요한 인터랙티브 요소(탭·필터·접힘·정렬·간단한 차트)만 보강**한다.

**동작:**
1. Phase 2 SEED LOAD에서 입력 톤·color를 추정해 `references/slide-templates/index.json` 35종 중 top-3 추천.
2. 사용자 confirm 또는 신뢰도 ≥ 0.85 시 top-1 자동 선택.
3. BLACK은 선택된 템플릿 HTML을 베이스로 받아:
   - 슬라이드/섹션 컨테이너에 콘텐츠를 H1/H2 단위로 주입
   - 입력 성격상 필요한 경우 인터랙션 보강 허용 (예: 여행 일정의 일자 탭, 식단표의 식사 필터, 학습 노트의 챕터 접힘)
   - 디자인 토큰(컬러·타이포·여백)은 템플릿의 것을 그대로 사용. 새로 만들지 않는다.
4. RGSB 채점은 generic 페르소나 프로파일 사용 (SKILL.md Phase 1 §5 참조).

**금지:**
- 템플릿 미선택 상태로 자체 디자인 처음부터 생성 (= `direct` 모드 영역 침범).
- 템플릿의 컬러·타이포·여백을 새 토큰으로 재정의.
- 산출물을 텍스트만 나열한 무디자인 페이지로 출력.
