# INTERNAL_CONTRADICTION — 자기 모순

## 검출 기준

다음 중 하나라도 BLACK 산출물에 나타나면 양성:

1. **라벨 ↔ 본문 충돌:** 같은 카드/슬라이드/단락의 라벨(area·카테고리·태그)이 본문 설명(desc·주소·운영시간·노트)과 불일치.
   - 예: 호텔 카드 area "신사이바시 · 프리미엄"인데 desc 본문은 "우메다 그랜드 프론트 직결".
2. **동일 객체 수치 불일치:** 같은 객체의 가격·시간·인원·거리가 서로 다른 위치에서 다르게 표기됨.
   - 예: 슬라이드 3에서 "라피트 1,490엔" / 슬라이드 6에서 "라피트 1,290엔" 같은 객체인데 가격 다름 + 차이 설명 없음.
3. **Pre-Processing Pack 충돌:** `pre-processing.json`의 `dates_resolved`·`param_defaults`와 산출물 본문이 모순.
   - 예: Pack에 "start: 2026-05-22 (Fri)" 박혀 있는데 본문에서 "5/22 토요일"로 적힘.
   - 예: Pack에 "param_defaults.인원: 1인" 박혀 있는데 본문 어디서는 "2인 기준 가격"으로 적힘 + 라벨 갱신 없음.
4. **헤더 ↔ 본문 충돌:** 표지 또는 섹션 헤더가 약속한 분량·항목 수와 실제 본문 개수가 다름.
   - 예: 헤더 "다섯 가지 체크리스트"인데 실제 블록은 4개.
5. **날짜·요일 자가 모순:** 같은 산출물 안에서 같은 날짜의 요일을 다르게 표기.
   - 예: 표지 "5/22 (금)" / Day 1 슬라이드 "5월 22일 토요일".

## 검출하지 않는 것

- 의도된 대비·비교 (예: "보수 시나리오 100억 / 낙관 시나리오 150억" — 시나리오가 명시되어 다른 객체)
- 시점 차이가 명시된 가격 변동 (예: "정가 1,490엔 / 웹 사전구매 할인 1,290엔" — 두 가격 모두 출처와 함께 표기)
- BLACK이 의도적으로 옵션을 병렬 제시 (예: 호텔 A/B/C 가격대 분포)
- 사용자 입력 자체가 모순일 때 BLACK이 그 모순을 명시적으로 지적한 경우

## 양성 예시 5

1. 호텔 카드 area "신사이바시 · 프리미엄" + desc "우메다 그랜드 프론트 직결" → area와 위치 본문 충돌.
2. 표지 "5/22 (금) · 5/23 (토) · 5/24 (일)" + Day 1 헤더 "5월 22일 · 토요일 · Arrival" → 요일 자가 모순.
3. 항공권 카드 "왕복 약 35~50만원" + 다른 슬라이드의 같은 옵션 "왕복 약 25~40만원" → 가격 불일치, 시점·조건 차이 명시 없음.
4. 체크리스트 헤더 "다섯 가지만 챙기면 됩니다" + 실제 블록 4개 → 헤더 약속과 본문 불일치.
5. Pre-Processing Pack에 "param_defaults.출발지: ICN" + 본문 어딘가 "김해공항 출발 기준" → Pack 충돌.

## 음성 예시 5

1. 호텔 카드 area "우메다 · 프리미엄" + desc "우메다 그랜드 프론트 직결" → 라벨과 본문 정합 ✓
2. 항공권 옵션 A "LCC 18~25만원" / 옵션 B "FSC 35~50만원" → 다른 옵션, 의도된 병렬 ✓
3. "라피트 정가 1,490엔, 웹 사전구매 외국인 할인 1,290엔 별도" → 두 가격 차이 명시 ✓
4. "네 가지 블록 추가" + 실제 블록 4개 → 헤더와 본문 정합 ✓
5. Pack "dates_resolved.start: 2026-05-22 (Fri)" + 본문 "5월 22일 금요일 출발" → Pack 정합 ✓

## BLACK 재작성 지시 템플릿

```
직전 산출물에 INTERNAL_CONTRADICTION 안티패턴이 검출되었습니다:

검출된 위치:
- {위치 A}: "{표기 A}"
- {위치 B}: "{표기 B}"
- 충돌 유형: {label_body | numeric | preprocessing_pack | header_body | date_dayofweek}

재작성 지시:
1. 두 표기 중 어느 쪽이 정합한지 1차 출처 또는 Pre-Processing Pack을 기준으로 판정하세요.
2. 정합한 쪽으로 통일하거나, 두 표기가 모두 유효한 경우(가격 정가/할인 등) 차이 사유를 명시하세요.
3. Pre-Processing Pack과 충돌이면 Pack을 신뢰하고 본문을 수정하세요. Pack이 틀렸다고 판단되면 reasoning.md에 사유 기록 후 Pack을 갱신하세요.
4. 라운드 카운트는 변동 없습니다.
```

## 적용 제외

- 카테고리 `소설 쓰기`(c63)·`시 쓰기`(c64) — 의도된 자기 모순(아이러니·반전)이 문학적 장치인 경우.
- 카테고리 `리더 심리 상담`(c68) — 사용자 내적 모순을 그대로 반영하는 것이 본질.
